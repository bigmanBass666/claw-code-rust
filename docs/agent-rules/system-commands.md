# 系统命令（斜杠命令协议）

> 系统级协议，不属于任何单一 Agent。任何 AI 会话收到 `/` 开头的输入都必须遵循本协议。
>
> **唯一入口**：`/` 前缀 = 命令模式。AGENTS.md 负责信号检测，本文件负责注册、匹配与执行。

---

## 📡 斜杠命令协议

### 触发条件（铁律）

用户输入以 `/` 开头时，进入命令模式。无例外。

### 命令注册表

| 命令 | 别名 | 类别 | 匹配关键词 | 说明 |
|------|------|------|-----------|------|
| `/status` | `/s`, `/状态` | 信息 | 状态, 进度, status, 当前 | 查看系统状态 |
| `/reset` | `/r`, `/重置` | 操作 | 重置, reset, 从头开始, 清理 | 系统重置（需确认） |
| `/audit` | `/a`, `/审计` | 审计 | 审计, audit, 一致性, 检查 | 审计系统一致性 |
| `/help` | `/h`, `/?` | 信息 | 帮助, help, 命令, 列表 | 列出所有可用命令 |
| `/log` | `/l`, `/日志` | 信息 | 日志, log, 历史, 流水账 | 查看命令历史 |

### 协议流程

1. **检测** — 输入以 `/` 开头 → 进入命令模式
2. **提取** — 取 `/` 后的第一个词作为候选命令名
3. **精确匹配** — 在注册表中查找命令名或别名 → 命中则执行
4. **模糊匹配** — 未命中时，对 `/` 后的完整文本匹配关键词字段 → 高置信度直接执行，低置信度输出候选列表让用户确认
5. **未知命令** — 完全无法匹配 → 输出 `/help` 内容 + 提示
6. **记录** — 执行完毕后追加到 `tasks/logs/system-commands.log`

### 兼容层（过渡期）

如果用户没有使用 `/` 前缀，但输入包含明确的系统命令关键词（如"执行系统重置""帮我审计"），仍可触发对应命令。
→ 触发时提示：``💡 建议使用斜杠格式，如 `/reset` 或 `/audit`，更精准且可扩展。``
→ 此兼容层将在未来版本移除。

## 📋 命令详情

### 1. /status — 查看状态

#### 执行步骤

1. 读取 `tasks/shared/agent-status.md` — Agent 状态 + 当前迭代任务看板
2. 读取 `tasks/shared/iteration-log.md` — 当前迭代字段（状态/任务数/已完成）
3. 运行 `git status` — Git 状态

#### 输出格式（必须）

```markdown
## 当前状态

### 迭代状态: Iteration [N]
- **状态**: [进行中/已完成/已废弃/待开始]
- **任务**: [X] 个任务，[Y] 已完成
- **当前任务**: [进行中任务的描述，如无则写"无"]

### Agent 状态
[从 agent-status.md 表格中读取]

### Git 状态
- **分支**: [当前分支]
- **未暂存改动**: [数量] 个（[列出文件名]）
- **未追踪文件**: [如有]

### 下一步建议
[如果迭代进行中 → "请唤醒 [等待中的Agent]"；如果待开始 → "请唤醒 Planner 制定新计划"]
```

#### ⚠️ 准确性铁律

- 迭代状态**必须与实际一致**
- Agent 状态从 `agent-status.md` 如实读取，不推测、不编造

### 2. /reset — 系统重置

#### 执行协议

详见 `cli-operations.md#系统重置`

#### 确认步骤

实际修改文件前，输出确认清单并等待用户回复：

```
⚠️ 即将执行完全重置，将执行以下操作：
  1. 清空所有 inbox（7个文件）
  2. 重置 agent-status.md 所有Agent为未启动
  3. 归档 Iteration X → 新建 Iteration X+1
  4. 清空 queue.md/assignments.md/observations.md 等

确认执行？回复"确认"开始。
```

#### 输出格式（执行完成后）

```markdown
## 重置完成

- **操作类型**: 完全重置
- **归档迭代**: Iteration [X] → 已废弃
- **新迭代**: Iteration [X+1] 就绪
- **已清空**: inbox（7个）/ queue.md / observations.md / agent-status.md

请唤醒 Planner 开始新迭代。
```

### 3. /audit — 审计系统

#### 执行步骤

1. 触发 `valveos-audit` skill
2. 等待审计结果

#### 输出格式

```markdown
## 审计结果

- **P0 问题**: [X] 个 — [描述]
- **P1 问题**: [X] 个 — [描述]
- **通过项**: [列出]

[如有问题 → 修复建议]
[如全部通过 → "✅ 系统一致性检查通过"]
```

---

### 4. /help — 命令帮助

#### 执行步骤

输出格式化的命令列表。

#### 输出格式

```markdown
## 可用命令

| 命令 | 别名 | 说明 | 示例 |
|------|------|------|------|
| /status | /s, /状态 | 查看系统状态 | `/status` 或 `/看看状态` |
| /reset | /r, /重置 | 系统重置（需确认） | `/reset` 或 `/重置一下` |
| /audit | /a, /审计 | 审计系统一致性 | `/audit` 或 `/帮我审计` |
| /log | /l, /日志 | 查看命令历史 | `/log` |
| /help | /h, /? | 显示本帮助 | `/help` |

使用方式：直接输入 `/命令名`，也支持 `/+自然语言` 如 `/重置一下系统`
```

---

### 5. /log — 命令历史

#### 执行步骤

1. 读取 `tasks/logs/system-commands.log`
2. 展示最近 20 条记录（或全部如果不足 20 条）
3. 如果文件为空或不存在，提示"暂无命令记录"

#### 输出格式

```markdown
## 命令历史（最近 N 条）

[从 system-commands.log 读取并展示]

---
💡 提示：完整日志文件位于 tasks/logs/system-commands.log
```
