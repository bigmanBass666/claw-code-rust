# 系统命令

> 系统级协议，不属于任何单一 Agent。任何 AI 会话（包括非 ValveOS Agent）收到这些命令都必须遵循统一执行协议。
>
> **分层设计**：AGENTS.md 路由表负责**触发**（模糊匹配），本文件负责**执行**（精确定义）。AI 读到本文件 = 已被触发。

---

## 📋 命令总览

| 指令 | 类别 | 特性 |
|------|------|------|
| 查看状态 | 信息类 | 只读不写，按统一格式输出 |
| 系统重置 | 操作类 | 修改系统状态，需二次确认 |
| 审计系统 | 审计类 | 检查一致性，输出问题清单 |

---

## 1. 查看状态

### 执行步骤

1. 读取 `tasks/shared/agent-status.md` — Agent 状态 + 当前迭代任务看板
2. 读取 `tasks/shared/iteration-log.md` — 当前迭代字段（状态/任务数/已完成）
3. 运行 `git status` — Git 状态

### 输出格式（必须）

```markdown
## 当前状态

### 迭代状态: Iteration [N]
- **状态**: [进行中/已完成/已废弃/待开始]
- **任务**: [X] 个任务，[Y] 已完成
- **当前任务**: [进行中任务的描述，如无则写"无"]

### Agent 状态
[从 agent-status.md 表格中读取]

### Git 状态
- **分支**: [当前分支]
- **未暂存改动**: [数量] 个（[列出文件名]）
- **未追踪文件**: [如有]

### 下一步建议
[如果迭代进行中 → "请唤醒 [等待中的Agent]"；如果待开始 → "请唤醒 Planner 制定新计划"]
```

### ⚠️ 准确性铁律

- 迭代状态**必须与实际一致**。进行中不要说"刚完成"，已废弃不要说"就绪"，待开始不要说"进行中"
- Agent 状态从 `agent-status.md` 如实读取，不推测、不编造

## 2. 系统重置

### 精确触发词（必须匹配）

- `执行系统重置`
- `完全重置系统`

> ⚠️ 只有上述精确短语才直接触发。路由表中列出的模糊词（"重置状态""重置一下""清理一下"）需先确认意图。

### 执行协议

详见 `cli-operations.md#系统重置`

### 确认步骤

实际修改文件前，输出确认清单并等待用户回复"确认"/"是"/"yes"/"执行"：

```
⚠️ 即将执行完全重置，将执行以下操作：
  1. 清空所有 inbox（7个文件）
  2. 重置 agent-status.md 所有Agent为未启动
  3. 归档 Iteration X → 新建 Iteration X+1
  4. 清空 queue.md/assignments.md/observations.md 等

确认执行？回复"确认"开始。
```

### 输出格式（执行完成后）

```markdown
## 重置完成

- **操作类型**: 完全重置
- **归档迭代**: Iteration [X] → 已废弃
- **新迭代**: Iteration [X+1] 就绪
- **已清空**: inbox（7个）/ queue.md / observations.md / agent-status.md

请唤醒 Planner 开始新迭代。
```

## 3. 审计系统

### 执行协议

触发 `valveos-audit` skill

### 输出格式

```markdown
## 审计结果

- **P0 问题**: [X] 个 — [描述]
- **P1 问题**: [X] 个 — [描述]
- **通过项**: [列出]

[如有问题 → 修复建议]
[如全部通过 → "✅ 系统一致性检查通过"]
```
